# Internationalization (i18n) & RTL

Atlas ships in **English** and **Arabic** with full **right-to-left** support, built on
[next-intl](https://next-intl.dev) with **locale-prefixed routing**.

## Routing

- Every page lives under `app/[locale]/…`; URLs are prefixed: `/en/products`, `/ar/products`.
- `proxy.ts` (Next.js 16's renamed middleware) runs `next-intl`'s middleware to detect the locale and
  redirect `/` → `/en`. It is nodejs-runtime and excludes `/api`, `/_next`, and static files via its matcher.
- `src/i18n/routing.ts` defines `locales: ['en','ar']`, `defaultLocale: 'en'`, `localePrefix: 'always'`.
- `src/i18n/navigation.ts` exports locale-aware `Link`, `useRouter`, `usePathname`, `redirect`,
  `getPathname` — used everywhere instead of `next/link` / `next/navigation` so the prefix is automatic.
- API routes stay at the top level (`app/api`) — they are not localized.

## Messages

- All user-facing strings live in `messages/en.json` and `messages/ar.json`, organized by namespace
  (`nav`, `footer`, `auth`, `register`, `dashboard`, `management`, `users`, `account`, `browse`,
  `home`, `about`, `contact`, `legal`, `catalogue`, `common`, `errors`, `metadata`, `language`).
- Server components & `generateMetadata` use `getTranslations`; client components use `useTranslations`.
- **No user-facing string is hardcoded** — including buttons, placeholders, validation/empty states,
  toasts, badges, and long-form legal/marketing prose (all translated to Arabic).
- API **error codes** (`CONFLICT`, `RATE_LIMITED`, …) map to translated messages via
  `src/lib/useErrorMessage.ts`, so server errors surface localized rather than raw English.

## Formatting

- Dates/numbers use next-intl's `useFormatter`/`getFormatter` (locale-aware; Arabic numerals & formats).

## RTL

- `app/[locale]/layout.tsx` sets `<html lang={locale} dir={locale==='ar'?'rtl':'ltr'}>`.
- Components use **logical** Tailwind utilities (`ps/pe`, `ms/me`, `start/end`, `text-start/end`,
  `rounded-s/e`) so layouts mirror automatically; directional icons (pagination chevrons) flip by locale.
- Sonner toasts and shadcn `Sheet` respect direction.
- The Arabic font (Cairo, via `next/font`) is applied through `html[lang="ar"]` in `globals.css`.

## SEO

Each page advertises its locale variants via `alternates.languages` (hreflang) and a per-locale
`canonical`; `sitemap.ts` emits both `/en` and `/ar` URLs. See [SEO.md](./SEO.md).

## Adding a language

1. Add the code to `routing.locales`.
2. Add `messages/<code>.json`.
3. If RTL, add it to the `dir` check in `app/[locale]/layout.tsx` and (optionally) a suitable font.
